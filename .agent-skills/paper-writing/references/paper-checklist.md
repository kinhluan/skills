# Paper Writing Checklist

## Pre-Writing

- [ ] Target venue identified (conference/journal)
- [ ] Venue format downloaded (LaTeX template, page limits)
- [ ] Key papers from venue reviewed (2-3 recent publications)
- [ ] Contribution statement clear (1-2 sentences)

## Structure Checklist

### Title
- [ ] Descriptive, includes key contribution
- [ ] Within word limit (usually 10-15 words)
- [ ] No jargon without context

### Abstract (150-250 words)
- [ ] Problem statement (1-2 sentences)
- [ ] Gap/limitation of existing work (1 sentence)
- [ ] Our approach (2-3 sentences)
- [ ] Key results with numbers (1-2 sentences)
- [ ] Impact/significance (1 sentence)

### Introduction
- [ ] Context and motivation (why this matters)
- [ ] Problem definition (clear, specific)
- [ ] Limitations of existing approaches
- [ ] Our contribution (numbered list)
- [ ] Paper organization (last paragraph)

### Related Work
- [ ] Organized thematically, not chronologically
- [ ] Positions our work relative to existing
- [ ] Includes recent work (last 2-3 years)
- [ ] Identifies gap our work fills

### Methodology
- [ ] Problem formalization (math notation)
- [ ] Algorithm/system description
- [ ] Complexity analysis (time/space)
- [ ] Assumptions clearly stated
- [ ] Figures for complex concepts

### Experiments
- [ ] Research questions stated
- [ ] Datasets described (size, source, preprocessing)
- [ ] Baselines selected (recent, relevant)
- [ ] Metrics defined (primary bolded)
- [ ] Implementation details (hardware, hyperparameters)
- [ ] Results with mean ± std (min 3 seeds)
- [ ] Statistical significance tests
- [ ] Ablation studies
- [ ] Analysis (why method works/fails)

### Discussion
- [ ] Interpretation of results
- [ ] Limitations acknowledged
- [ ] Threats to validity

### Conclusion
- [ ] Summary of contributions
- [ ] Key findings
- [ ] Future work

## Post-Writing

- [ ] All figures readable in grayscale
- [ ] All tables have captions
- [ ] References formatted correctly
- [ ] Page limits respected
- [ ] Supplementary material prepared
- [ ] Code/data availability statement
